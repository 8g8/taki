                Motivation␊
              </h2><p><a href="https://github.com/yeoman/yo" target="_blank">yeoman</a> is too complex, and <a href="https://github.com/vuejs/vue-cli" target="_blank">vue-cli</a> has a narrow name, so I made <strong>sao</strong>.</p>␊
    <p><strong>sao</strong> is compatible with:</p>␊
    <ul>␊
    <li>regular git repo</li>␊
    <li>sao template as git repo</li>␊
    <li>sao template as npm package</li>␊
    <li>sao template in local folder</li>␊
    </ul>␊
    <h2 id="install" class="markdown-heading markdown-toc-heading">␊

              </h2><pre data-lang="bash"><code class="lang-bash">yarn global add sao␊
    <span class="token comment" spellcheck="true"># npm i -g sao</span>␊
    </code></pre>␊

                From Repo␊
              </h3><p>You can generate project from a git repo:</p>␊
    <pre data-lang="bash"><code class="lang-bash"><span class="token function">cd</span> new-project␊
    <span class="token comment" spellcheck="true"># for a regular git repo</span>␊
    <span class="token comment" spellcheck="true"># it copies all contents in the template to current working directory</span>␊
    sao user/repo␊
    </code></pre>␊

                From Package␊
              </h3><p>You can also generate project from an npm package:</p>␊
    <pre data-lang="bash"><code class="lang-bash"><span class="token comment" spellcheck="true"># if the name does not contains slash `/`</span>␊
    <span class="token comment" spellcheck="true"># it will be recognized as an npm package and prefixed with `template-`</span>␊
    <span class="token comment" spellcheck="true"># and you should install it before running `sao`</span>␊
    yarn global add template-vue␊
    <span class="token function">cd</span> new-project␊
    sao vue␊
    </code></pre>␊
